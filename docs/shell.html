Connect

Start a shell session on a remote host.

Shell
onyx user@host

Session behavior

Short drops reconnect automatically. tmux keeps the remote shell state on the server.

Reconnect

Brief transport loss triggers reconnect instead of a fresh login.

tmux-backed

Shell state and scrollback stay on the remote side.

Fallback

Onyx prefers QUIC and falls back to SSH when needed.

Built for real network conditions.

Packet loss, flaky Wi-Fi, and blocked UDP are normal. Onyx is built to keep the remote side running and recover cleanly.

Handles packet loss Keeps the client focused on reconnect and resume instead of forcing you to restart work.
Works over flaky Wi-Fi Interactive sessions and exec jobs are designed for short drops, not perfect links.
Falls back to SSH automatically Prefer QUIC when UDP works. Use SSH when it does not.
Keeps jobs running server-side onyx exec jobs keep going on the remote even after the client disconnects.

Stable interactive sessions.

Use Onyx like a normal shell target. It reconnects automatically, keeps the session under tmux, and retains it for up to 12 hours by default.

Interactive shell 
onyx user@host

Start a remote shell with QUIC first, automatic SSH fallback, and tmux-backed session state on the server.

Reconnects automatically Short network drops trigger reconnect instead of forcing a fresh login.
tmux-backed Keep shell state and scrollback on the server instead of treating the session as disposable.
12h retention The remote tmux session is retained for up to 12 hours by default.

Run commands that don’t die.

onyx exec starts a server-side job. Disconnect, reattach, read logs, or list jobs later. Add --json when automation needs structured output.

onyx exec
status starting gpu-box • job_84f31
$ onyx exec gpu-box --detach -- python train.py
[job] job_84f31 running detached
train epoch=01 loss=0.912 samples=4096
train epoch=02 loss=0.734 samples=8192
[net] connection lost...
[job] resumed seq=27
train epoch=03 loss=0.581 samples=12288
train epoch=04 checkpoint=best.pt
Run onyx exec prod -- deploy.sh

Start a remote command and stay attached to its output.

Detach onyx exec gpu-box --detach -- python train.py

Start a job and get back a job id immediately.

Resume onyx attach gpu-box <job_id>

Reattach after a disconnect or later from the same target.

Logs onyx logs gpu-box <job_id>

Read buffered output without taking over the running job.

onyx jobs gpu-box --json emits NDJSON job records for scripts, CI, and AI agents.

More than a shell.

Run commands that survive the network. onyx exec makes remote execution resumable, inspectable, and scriptable — useful for deploys, training runs, and AI-driven tooling that can't afford to lose state to a flaky Wi-Fi hop.

  • Resumable. The server owns the child process; your laptop can sleep, reconnect, or die — the job keeps running.
  • Reattachable. onyx attach replays the ring buffer and streams live output. onyx logs snapshots what's captured.
  • Detachable. --detach prints the job id and exits. Pick it up again from any machine.
  • Scriptable. --json emits one NDJSON event per line — clean input for CI pipelines and AI agents.
onyx exec prod -- deploy.sh onyx exec gpu-box --detach -- python train.py onyx attach gpu-box job_a1b2c3d4 onyx logs gpu-box job_a1b2c3d4
onyx exec running
$ onyx exec gpu-box --detach -- python train.py

Built for real remote work

QUIC + SSH fallback Prefer QUIC. Fall back cleanly when the network says otherwise.
Session persistence tmux-backed sessions with smoother reconnect behavior and scrollback.
Port forwarding Forward local ports to remote services through the same Onyx connection.
SSH ProxyCommand support Use Onyx as a transport layer for SSH-based tools and existing configs.
Resumable remote exec onyx exec runs commands as jobs that survive client disconnects. Reattach, tail, or capture JSON events later.
AI-friendly workflows Structured --json output and long-lived job ids fit cleanly into agent loops and CI pipelines.

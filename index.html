Built for real remote work

Reliable on flaky networks QUIC first. SSH fallback when needed.
Session persistence tmux-backed sessions with cleaner reconnect behavior.
Port forwarding Forward local ports to remote services through the same connection.
SSH tool compatibility Use Onyx as a transport layer for SSH-based workflows.

Fits existing workflows

Drop-in SSH transport for existing workflows.

SSH config 
Host dev-onyx
  HostName server
  ProxyCommand onyx proxy %h %p
Port forwarding 
onyx --forward 8888:8888 user@server
Shell session 
onyx user@server

How it works

01 Bootstrap over SSH

Upload and start the remote server automatically.

02 Connect over QUIC

Use the faster path when UDP is available.

03 Fall back automatically

Switch to SSH when UDP is blocked or unavailable.

Install

curl -sSL https://useonyx.dev/install.sh | bash
